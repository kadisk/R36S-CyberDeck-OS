# Resultado — Fase 3 (tela + input no R36S físico)

**Data:** 2026-06-15
**Resultado:** ✅ **UI NATIVA NA TELA, NAVEGÁVEL PELO JOYPAD** — confirmado pelo usuário.

## Marcos atingidos

- **Tela:** painel 640×480 **32bpp** (stride 2560), `/dev/fb0` + `/dev/dri/card0`.
  O renderizador `cyberdeck-fb` desenha a UI direto no framebuffer (sem GL).
- **Input:** joypad `odroidgo3-joypad` (`/dev/input/event1`, "GO-Super Gamepad")
  lido e mapeado; **navegação por D-pad funciona** no aparelho.
- **Backlight:** ajustável por **L2/R2** (`/sys/class/backlight/backlight/brightness`,
  máx 160).
- **Bateria:** `/sys/class/power_supply/{battery,ac,usb}` — capacidade/status no STATUS.

## Dados de hardware coletados (referência)

- `docs/hardware/device-captures/cyberdeck-fbinfo.txt` — fb/DRM/input/backlight.
- `docs/hardware/device-captures/joypad-capture-2026-06-15.txt` — mapa real dos botões.

## Como reproduzir

1. `scripts/create-test-sd-image.sh` → grava `r36s-cyberdeck-minimal.img`.
2. `sudo dd ... of=/dev/sdX` (microSD de teste).
3. Boot: a UI aparece no `tty1`; D-pad navega, L2/R2 brilho, F5 sai p/ shell.
